# site-watch

Скрипт, который следит, не поменялась ли страница на сайте, и пишет об этом в Discord через webhook. Делал, чтобы не проверять руками страницу с расписанием/ценами каждый день.

Без внешних API-ключей — только `requests` и webhook Discord (создаётся в настройках канала за 10 секунд).

## Как использовать

1. Список страниц для отслеживания — обычный текстовый файл:

```
# sites.txt
https://example.com
https://example.com/changelog
```

2. Одна проверка:

```bash
pip install -r requirements.txt
python -m sitewatch.watcher --urls sites.txt --webhook https://discord.com/api/webhooks/...
```

3. Непрерывное слежение (например, в cron или systemd не нужен — сам крутит цикл):

```bash
python -m sitewatch.watcher --urls sites.txt --webhook $DISCORD_WEBHOOK_URL --loop --interval 300
```

При первом запуске скрипт просто запоминает текущее состояние страницы — уведомление придёт только при следующем реальном изменении.

## Как это работает

- `sitewatch/diffing.py` — вырезает `<script>`/`<style>`, снимает теги, схлопывает пробелы и хэширует получившийся текст (SHA-256). Сравниваются не байты HTML, а видимый текст — так вёрстка/аналитика/рандомные атрибуты не считаются "изменением".
- `sitewatch/storage.py` — хранит последний хэш каждого URL в JSON-файле.
- `sitewatch/watcher.py` — обходит список, сравнивает хэши, шлёт вебхук при отличии.

## Тесты

```bash
pip install requests pytest
pytest tests/ -v
```

## Ограничения

- Сравнение по хэшу текста — не покажет **что** именно изменилось, только факт изменения. Для диффа придётся хранить сам текст, а не хэш.
- Не подходит для страниц, которые рендерятся JS после загрузки (SPA) — `requests` видит только исходный HTML.
- JSON-хранилище — нормально для десятков URL, не для тысяч.
